# Фича: Live-отслеживание автобуса на карте

**Ветка:** `04-live-tracking`
**Создана:** 2026-03-25
**Статус:** не начат

## Контекст

**Зачем:** У нового перевозчика есть live-отслеживание автобусов на сайте «пассажир-онлайн», но напрямую с фронта обратиться нельзя (CORS). Нужен прокси на бэкенде. Карта на фронте уже есть (Leaflet/MapTiler) — нужно просто добавить маркер автобуса.

**Что есть сейчас:** Карта с остановками в `src/widget/Map/`. Leaflet + MapTiler SDK. Live-данные перевозчика блокируются CORS при запросе с клиента.

## User Stories

### US-1 — Прокси live API перевозчика (Приоритет: P1)

Бэкенд проксирует запросы к API перевозчика, обходя CORS. Отдаёт позиции автобусов в удобном формате.

**Почему этот приоритет:** Без прокси live-данные недоступны на клиенте.

**Как проверить независимо:** GET `/api/live` → JSON с координатами автобусов.

**Сценарии приёмки:**

1. **Дано** автобусы на маршруте, **Когда** GET `/api/live`, **Тогда** ответ с массивом `[{ lat, lng, direction, speed, timestamp }]`
2. **Дано** API перевозчика недоступен, **Когда** GET `/api/live`, **Тогда** ответ 503 с `{ error: "Данные временно недоступны" }`
3. **Дано** бэкенд кеширует ответ, **Когда** два запроса за 5 секунд, **Тогда** второй возвращает кеш (не дёргает перевозчика каждый раз)

---

### US-2 — Автобус на карте (Приоритет: P1)

На существующей карте отображается маркер(ы) автобуса с live-позицией. Обновляется каждые 10-15 секунд.

**Почему этот приоритет:** Главный запрос пользователей — видеть где автобус прямо сейчас.

**Сценарии приёмки:**

1. **Дано** live-данные доступны, **Когда** пользователь смотрит карту, **Тогда** видит маркер автобуса на его текущей позиции
2. **Дано** live-данные доступны, **Когда** проходит 15 секунд, **Тогда** позиция маркера обновляется
3. **Дано** live-данные недоступны, **Когда** пользователь смотрит карту, **Тогда** маркер автобуса не отображается (нет ошибки, просто нет маркера)

---

## Ключевые сущности

- **BusPosition**: `lat`, `lng`, `direction` (куда едет), `speed`, `timestamp`, `route` (номер маршрута если есть)

## Edge Cases

- Что если перевозчик сменил API или его заблокировал? → Логировать ошибки, показывать в мониторинге, на фронте — graceful degradation
- Что если автобус стоит на месте 30 минут? → Показывать как есть, но можно добавить label «стоит»
- Сколько автобусов на маршруте одновременно? → [УТОЧНИТЬ] Нужно исследовать API перевозчика
- Какой формат данных у API перевозчика? → [УТОЧНИТЬ] Нужно проснифферить запросы на сайте перевозчика

## Критерии приёмки

- [ ] GET `/api/live` отдаёт позиции автобусов
- [ ] Бэкенд кеширует ответ перевозчика (TTL 5-10 сек)
- [ ] Маркер автобуса на карте обновляется каждые 10-15 сек
- [ ] Graceful degradation: если live недоступен — карта работает без маркера
